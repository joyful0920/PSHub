- **カテゴリ**: 標準入力 / 最小値の更新（タイブレークあり）

## 問題概要

N 個のお菓子を分配する。機械 i はお菓子を m_i 個の容器に均等に分け、余りを排出する。
**余りが最も少ない**機械の番号（1 始まり）を出力する。余りが同じ機械が複数あれば、
**容器の数が多い**方を優先する。m_i はすべて相異なる。

## 入力フォーマット

```
M N
m_1 .. m_M   ← 各機械の容器数（M 行）
```

## 制約

- 2 ≦ M ≦ 100、M ≦ N ≦ 1000、1 ≦ m_i ≦ N、m_i は相異なる

## 入出力例

入力:
```
3 7
3
6
5
```
出力:
```
2
```
（余りは 1, 1, 2。同点の 3 と 6 では容器数が多い 6 = 機械 2 を優先）

入力:
```
3 10
3
4
5
```
出力:
```
3
```
（余り 1, 2, 0 → 余り 0 の機械 3）

## 解法メモ

- **ベスト更新パターン**: 1 台目で `bestIndex` / `bestRemainder` を初期化し、
  2 台目以降を順に比較して更新する（番兵値が不要になる）。
- 更新条件を 1 つの if にまとめると仕様がそのまま読める:
  `余りがより小さい || (同点 && 容器数が多い)`。
- 配列のループに `for...in` は使わない（キーが文字列になる）。インデックスが要るなら
  普通の for 文。計算量 O(M)。
