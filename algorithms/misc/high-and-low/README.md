- **カテゴリ**: 標準入力 / 多重キー比較

## 問題概要

カードには番号が 2 つ書かれている（1 つ目は 1〜10、2 つ目は 1〜4）。強さのルールは:

1. **1 つ目の番号が大きい**カードが強い。
2. 1 つ目が同じなら、**2 つ目の番号が小さい**カードが強い（向きが逆な点に注意）。

親カード 1 枚と子カード n 枚が与えられるので、各子カードとの比較で親が強ければ `High`、
そうでなければ `Low` を 1 行ずつ出力する。同じカードは 2 枚存在しない。

## 入力フォーマット

```
a b      ← 親カード
n
A_i B_i  ← 子カード（n 行）
```

## 制約

- 1 ≦ a, A ≦ 10、1 ≦ b, B ≦ 4、1 ≦ n < 40、カードの重複なし

## 入出力例

入力:
```
5 3
3
8 1
2 4
5 6
```
出力:
```
Low
High
High
```
（8 には負け、2 には勝ち、1 つ目同点の 5 6 は 2 つ目が小さい親の勝ち）

入力:
```
2 4
2
2 1
2 6
```
出力:
```
Low
High
```

## 解法メモ

- 多重キー比較。1 キー目は「大きい方が強い」、2 キー目は「**小さい方**が強い」と向きが逆
  なので、分岐の不等号を逆にする。
- 完全同点（両方一致）は「同じカードは存在しない」制約で起きないので分岐不要。
- `||` チェーンで 1 行にもできる: `(a - first || second - b) > 0 ? "High" : "Low"`
  （2 キー目は向きが逆なので `second - b` と符号を反転）。計算量 O(n)。
