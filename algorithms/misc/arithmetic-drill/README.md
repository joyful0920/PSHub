- **カテゴリ**: 標準入力 / 構成的生成（重複なし）

## 問題概要

足し算 M 問、引き算 N 問の計算ドリルを作る。数値と答えはすべて 0〜99。同じ問題の
重複は不可（`a + b =` と `b + a =` は a≠b なら別問題として扱う）。条件を満たせば
並び順は自由。

## 入力フォーマット

```
M N
```

## 制約

- 0 ≦ M, N ≦ 5,050（それぞれ作成可能な最大数）、M + N ≧ 1

## 入出力例

入力:
```
5 3
```
出力例（条件を満たせば並びは自由）:
```
0 + 0 =
0 + 1 =
0 + 2 =
0 + 3 =
0 + 4 =
0 - 0 =
1 - 0 =
1 - 1 =
```

## 解法メモ

- 構成的に列挙する問題。**体系的に (a, b) を生成すれば重複は自然に起きない**ので
  Set は不要。
- 足し算: 結果が 0〜99 なので `a + b <= 99`。`a`、`b` を昇順に回して M 個で打ち切り。
- 引き算: 結果が 0 以上なので `a >= b`。同様に N 個で打ち切り。
- 生成可能な最大数はどちらも `100 + 99 + … + 1 = 5050`（制約の上限と一致）。
- ゼロ埋め不可なので数値をそのまま文字列化（`09` ではなく `9`）。計算量 O(M + N)。
