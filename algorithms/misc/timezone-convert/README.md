- **カテゴリ**: 標準入力 / 時刻・時差計算

## 問題概要

N 個の都市と、それぞれの UTC からの時差（時、負は遅れ）が与えられる。ある都市 q のユーザが
現地時刻 t（hh:mm）に投稿したとき、各都市のユーザの画面に表示される現地時刻を、
入力順に hh:mm 形式（0 埋め 2 桁）で出力する。24 時間制。

## 入力フォーマット

```
N
p_1 s_1
...
p_N s_N
q t
```

## 制約

- 1 ≦ N ≦ 100、-12 ≦ s_i ≦ 14、q は都市名のいずれか

## 入出力例

入力:
```
3
tokyo 9
paris 1
newyork -5
tokyo 12:00
```
出力:
```
12:00
04:00
22:00
```
（UTC = 12 - 9 = 3 時。paris: 3+1 → 04、newyork: 3-5 = -2 → 22）

## 解法メモ

- 3 段階：① 都市名 → オフセットの対応を作る、② 投稿都市のオフセットで UTC の時を求める
  (`hh - offset[q]`)、③ 各都市は `UTC + offset` を計算。
- **時計の巻き戻しは `Math.abs` ではなく正の剰余** `((x % 24) + 24) % 24`。
  例えば -5 時は 19 時であって 5 時ではない。`Math.abs(-5)=5` は誤り。
- オフセットは整数「時」なので分は変化しない。そのまま出力。計算量 O(N)。
