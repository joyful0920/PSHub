- **カテゴリ**: 標準入力 / シミュレーション（トーラス・正の剰余）

## 問題概要

上下端・左右端がつながった w×h のマップ上を、キャラクターが移動ログどおりに動く。
初期位置 (x, y)（(0,0) が左下）から N 回移動した後の位置を出力する。
`U`/`D`/`R`/`L` はそれぞれ上・下・右・左、後ろの数は移動距離。

## 入力フォーマット

```
w h n
x y
dir_i m_i   （n 行）
```

## 制約

- 1 ≦ w, h ≦ 1000、0 ≦ x < w、0 ≦ y < h、0 ≦ n ≦ 1000、0 ≦ m_i ≦ 1000

## 入出力例

入力:
```
7 6 1
3 4
U 3
```
出力:
```
3 1
```
（y: 4→5→0→1 と一周を挟む）

入力:
```
7 6 1
3 4
L 5
```
出力:
```
5 4
```

## 解法メモ

- 端がつながっているので座標は循環する。移動は素直に `+`/`-` し、**各移動後に
  正の剰余 `((v % size) + size) % size` で 0〜size-1 に收める**。
- **落とし穴**: `m_i` は最大 1000 で幅より大きくなり得るため、1 回の `±size` 補正では
  足りない（複数周する）。`w=1` のような極端な例でも剰余なら一発。`Math.abs` は誤り。
- 計算量 O(N)。
