- **カテゴリ**: 標準入力 / 桁の操作

## 問題概要

特殊フォントでは 0・1・8 は 180 度回転しても同じ、6↔9 は入れ替わり、2・3・4・5・7 は
回転すると数字にならない。A〜B の範囲で「180 度回転しても同じ数に読める」番号
（回転対称数）の個数を出力する。番号は 0 で始まらない。

## 制約

- 1 ≦ A ≦ B ≦ 100,000

## 入出力例

入力:
```
1 9
```
出力:
```
2
```
（1→1、8→8 の 2 個。6→9、9→6 は自分に戻らないので不可）

入力:
```
10 100
```
出力:
```
4
```
（11→11、69→69、88→88、96→96 の 4 個）

## 解法メモ

- 回転表 `ROT = {0:0, 1:1, 8:8, 6:9, 9:6}` を用意。
- 各数字を文字列にし、**逆順に**回転表を適用して回転後の文字列を作る。表にない数字が
  あれば即 `false`。
- 回転後の文字列が元と一致すれば回転対称。**数値化せず文字列で比較**すると、
  先頭ゼロ（`"01"` vs `"10"`）を自然に区別できる。
- A〜B を全探索（B ≦ 10 万なので O((B-A)×桁数) で十分)。
