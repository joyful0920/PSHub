- **カテゴリ**: 標準入力 / シミュレーション

## 問題概要

N 日間、1 日 1 回株価をチェックして次のルールで売買する。損益（初期資金からの増減）を出力する。

- 株価が `c1` 円以下 → 1 株買う
- 株価が `c2` 円以上 → 持ち株をすべて売る
- `c1` と `c2` の間 → 何もしない
- N 日目は上記を行わず、持ち株をすべて売る

損益はマイナスになることもある。

## 制約

- 1 ≦ N ≦ 100、1 ≦ c1 < c2 ≦ 1000、1 ≦ p_i ≦ 1000

## 入出力例

入力:
```
4 50 100
40
60
100
30
```
出力:
```
60
```
（40 で 1 株買い(-40)、60 は何もしない、100 で全売り(+100)、最終日 30 は持ち株 0）

入力:
```
3 50 90
45
40
20
```
出力:
```
-45
```
（45・40 でそれぞれ 1 株買い(-85)、最終日 20 で 2 株を全売り(+40) → 損益 -45）

## 解法メモ

- 状態は「持ち株数 `shares`」と「損益 `money`」の 2 つだけ。1 日ずつ順に更新する。
- 最終日は無条件で全売りなので `i === n - 1 || price >= up` を売却条件にまとめると分岐が減る。
  最終日は買わない仕様なので、この順序（売却を先に判定）が正しい。計算量 O(N)。
