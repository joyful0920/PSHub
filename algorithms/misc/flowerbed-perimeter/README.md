- **カテゴリ**: 標準入力 / 2 次元グリッド（周囲長）

## 問題概要

H×W の地図が `#`（花壇）と `.`（その他）で与えられる。花壇を過不足なく囲むのに必要な
ロープの本数（＝花壇領域の周囲長）を出力する。ロープの長さは区画の一辺と同じ。
花壇どうしが隣接する辺にはロープは不要。

## 入力フォーマット

```
H W
s_i   （'#'/'.' の長さ W の文字列、H 行）
```

## 制約

- 1 ≦ H, W ≦ 50

## 入出力例

入力:
```
2 2
##
#.
```
出力:
```
8
```
（L 字の 3 マス。外周をなぞると 8 辺）

入力:
```
1 3
#.#
```
出力:
```
8
```
（離れた 2 マスでそれぞれ 4 辺）

## 解法メモ

- 各 `#` 区画について、上下左右の隣が**花壇でない**（範囲外 or `.`）辺の数を数え、
  総和が周囲長。
- **範囲外を `false` 扱いする `isBed(r,c)` ヘルパー**を作ると、「外周」と「隣が `.`」を
  1 つの判定に統一できる。
- 方向は `dirs = [[-1,0],[1,0],[0,-1],[0,1]]`（上下左右）の**デルタ配列**で回すと、
  4 方向の重複コードが消える（グリッド探索の定石。BFS/DFS でも同じ形）。計算量 O(H×W)。
