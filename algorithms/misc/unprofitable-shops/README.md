- **カテゴリ**: 標準入力 / 条件カウント

## 問題概要

N 店舗を出店し M ヶ月営業する。1 店舗あたり建設費 A 円 + 毎月の人件費 B 円がかかり、
商品 1 つで C 円の利益が出る。各店舗の販売数が与えられるので、**投下費用以上の利益が
出ていない**（= 売上利益が費用を下回る）店舗の数を出力する。純利益ちょうど 0 の店は
閉店しない。

## 入力フォーマット

```
N M
A B C
R_1 .. R_N   ← 各店舗の販売数（N 行）
```

## 制約

- 1 ≦ N ≦ 1000、1 ≦ M ≦ 12、1 ≦ A, B ≦ 10000、1 ≦ C ≦ 500、1 ≦ R ≦ 1200

## 入出力例

入力:
```
3 2
500 250 10
99
100
150
```
出力:
```
1
```
（費用 = 500 + 250×2 = 1000。99×10 = 990 < 1000 のみ閉店。100×10 = 1000 はちょうどで存続）

入力:
```
2 1
100 100 1
50
60
```
出力:
```
2
```

## 解法メモ

- 費用 `A + B×M` は**ループ中に変わらない（ループ不変）**ので、外で 1 回だけ計算する。
  条件が `売上 < 費用` と読めるようになる副次効果もある。
- 「費用**以上**の利益が出ていない」の否定なので判定は **`<`（未満）**。ちょうど同額
  （純利益 0）は閉店しない — 以上／未満の境界は要件確認ポイント。計算量 O(N)。
