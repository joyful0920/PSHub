- **カテゴリ**: 標準入力 / シミュレーション / 多段入力

## 問題概要

N 人がボールを持ち、順番にパスしていく。各パスは「人 a が人 b に x 個渡す」と宣言する。
手持ちが x 個以上なら x 個を、足りなければ手持ちすべてを渡す。M 回のパスを処理した後、
各人の最終的な所持数を 1 行ずつ出力する。

## 入力フォーマット

```
N
s_1 .. s_N   （各人の初期所持数、N 行）
M
a b x        （パス情報、M 行）
```

## 制約

- 2 ≦ N ≦ 100、0 ≦ s_i ≦ 100、1 ≦ M ≦ 1000、1 ≦ x ≦ 100、a ≠ b

## 入出力例

入力:
```
3
5
5
5
1
1 3 2
```
出力:
```
3
5
7
```
（1→3 に 2 個 → [3, 5, 7]）

入力:
```
2
6
6
1
2 1 4
```
出力:
```
10
2
```
（2→1 に 4 個 → [10, 2]）

## 解法メモ

- 入力が「人数 → 初期所持 → パス数 → パス情報」の多段構成なので、`n + 1` / `n + 2` の
  オフセットで各セクションを `slice` する。
- 渡す数は「宣言 x」と「手持ち」の小さい方 → `Math.min(balls[a-1], x)`。
- パス情報は `for (const [a, b, x] of passes)` とヘッダで分割代入すると読みやすい。計算量 O(M)。
