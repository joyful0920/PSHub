- **カテゴリ**: 標準入力 / 集計・ソート

## 問題概要

N 人のエンジニアの名前と、M 件の書籍購入記録（誰がいくら使ったか）が与えられる。
各人の購入費合計を求め、高い順に名前を出力する。購入費が同額の社員は存在しない。

## 入力フォーマット

```
N
s_1 s_2 ... s_N   ← 全員の名前
M
o p              ← 購入者名 金額（M 行）
```

## 制約

- 1 ≦ N ≦ 100、N ≦ M+1 ≦ 100、1 ≦ 金額 ≦ 10000
- 各購入者は必ず名簿にいる。合計が同額の社員はいない。

## 入出力例

入力:
```
3
kim lee park
4
kim 500
lee 500
lee 2000
park 800
```
出力:
```
lee
park
kim
```
（lee 2500 / park 800 / kim 500）

入力:
```
3
a b c
2
b 500
c 300
```
出力:
```
b
c
a
```
（a は購入 0 円。それでも順位に含め、最下位になる）

## 解法メモ

- **落とし穴**: `N ≦ M+1` なので、購入記録に一度も出てこない社員（合計 0 円）が最大 1 名いる。
  購入記録だけを集計すると、その人が出力から漏れる。
- 対策: **まず名簿（2 行目）の全員を 0 で初期化**してから購入額を加算する。
- 集計は `Record<string, number>`（名前 → 合計）。最後に名前配列を合計の降順でソート。計算量 O(M + N log N)。
