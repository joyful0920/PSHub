- **カテゴリ**: 標準入力 / 文字列（部分一致カウント）

## 問題概要

1 月 1 日からの経過日数（0〜364 の 365 日）のうち、その日数の**文字列に数字 X が
含まれる**日を「幸運な日」とする。1 年で幸運な日が何日あるか数える。
1/1 は 0 日目で、これも対象。

## 制約

- 0 ≦ X ≦ 364

## 入出力例

入力:
```
7
```
出力:
```
63
```
（経過日数 0〜364 で "7" を含む日が 63 日）

入力:
```
99
```
出力:
```
3
```
（99, 199, 299 の 3 日）

## 解法メモ

- 経過日数 0〜364 を全探索し、`String(day).includes(x)` で部分一致を数える。計算量 O(365)。
- **落とし穴 1**: 0 日目を含めるので、ループは **`day = 0` から**。1 から始めると
  X=0 のとき 0 日目（"0"）を数え損ねて 1 ずれる。
- **落とし穴 2**: 部分一致は `includes` を使う。`match` は正規表現を取るため、意図
  （ただの部分文字列判定）とずれる（数字なら偶然動くが `includes` が正しい）。
