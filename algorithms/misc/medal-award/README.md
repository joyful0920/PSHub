- **カテゴリ**: 標準入力 / 順位カウント + 表引き

## 問題概要

N 人の競技者の点数が与えられる。各競技者について、**自分より高い点数の人数**に応じて
メダルを出力する。

- 0 人 → `G`（金）、ちょうど 1 人 → `S`（銀）、ちょうど 2 人 → `B`（銅）、それ以外 → `N`

同点者は同じメダルを受け取る（同点は「自分より高い」に数えない）。

## 入力フォーマット

```
N
s_1 s_2 ... s_N
```

## 制約

- 3 ≦ N ≦ 100、0 ≦ s_i ≦ 2000

## 入出力例

入力:
```
4
50 90 90 70
```
出力:
```
N
G
G
B
```
（90 が同点首位で 2 人とも G。70 は上に 2 人で B、50 は上に 3 人で N）

入力:
```
3
10 20 30
```
出力:
```
B
S
G
```

## 解法メモ

- **定義どおり「自分より高い人数」を数える**のが最も安全。`filter((q) => q > p).length` →
  人数からメダルを**表引き** `{0:"G", 1:"S", 2:"B"}[higher] ?? "N"`。
- 同点は `>` に入らないので、同点者は自動的に同じメダルになる — ソートして
  「前の順位の次のメダル」を配る方式は、**首位に同点が並ぶと 1 段ずれるバグ**を生む
  （例: `100 100 90` の 90 は上に 2 人いるので B が正解。順位チェーン式だと S になってしまう）。
- N ≦ 100 なので全員×全員の O(N²) で十分。
