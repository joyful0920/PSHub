- **カテゴリ**: 標準入力 / シミュレーション（区間更新 + 上限クランプ）

## 問題概要

一列に並んだ N 人に豆を配る。命令は「A 番目から B 番目までの全員に C 個ずつ配る」という
形式で M 個与えられる。ただし各人は**自分の年齢を超える個数は受け取らない**（年齢が上限）。
全命令の実行後、各人が持っている豆の数を 1 行ずつ出力する。

## 入力フォーマット

```
N
Y_1 .. Y_N   ← 年齢（N 行）
M
A B C        ← 命令（M 行）
```

## 制約

- 1 ≦ N ≦ 1000、1 ≦ Y ≦ 100、1 ≦ M ≦ 100、1 ≦ A ≦ B ≦ N、1 ≦ C ≦ 100

## 入出力例

入力:
```
2
5
30
2
1 2 4
1 2 4
```
出力:
```
5
8
```
（1 人目: 4 → 上限 5 で止まる。2 人目: 4 + 4 = 8）

入力:
```
3
2
3
10
1
1 3 3
```
出力:
```
2
3
3
```

## 解法メモ

- 状態は各人の所持数の配列。命令ごとに区間 `[A, B]` を回して加算する。
- 「年齢を超えない」は**上限クランプ** `Math.min(totals[i] + c, ages[i])` の 1 行で表せる
  （すでに上限に達していても min がそのまま保つので、事前の分岐は不要）。
- N ≦ 1000・M ≦ 100 なので、区間を素直に二重ループで回して O(N×M) で十分。
