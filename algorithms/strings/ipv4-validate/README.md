- **カテゴリ**: 文字列 / バリデーション

## 問題概要

`.` と数字だけからなる文字列が M 個与えられる。各文字列が IPv4 アドレスとして正しいか判定し、
正しければ `True`、違えば `False` を出力する。IPv4 は「`.` 区切りの 4 つの 10 進数、
各数は 0〜255」。

## 入力フォーマット

```
M
（アドレス文字列を M 行）
```

## 制約

- 1 ≦ N（各文字列長）≦ 100、1 ≦ M ≦ 100
- 入力は `.` と数字のみで構成される

## 入出力例

入力:
```
4
192.400.1.10.1000...
4.3.2.1
0..33.444...
1.2.3.4
```
出力:
```
False
True
False
True
```

## 解法メモ

- `split(".")` して **ちょうど 4 セグメント**であることを確認。
- **空セグメントを filter で消してはいけない**（`"1..2.3.4"` や `"1.2.3.4."` が誤って通る）。
  連続・先頭・末尾のドットは「空文字のセグメント」として現れるので、`part === ""` を明示的に弾く。
- 各セグメントを数値化し 0〜255 の範囲チェック。計算量 O(文字列長)。
- ※本問は「範囲 0〜255」しか指定がないので、先頭ゼロ（`"01"`）の可否は範囲判定のみで扱う。
